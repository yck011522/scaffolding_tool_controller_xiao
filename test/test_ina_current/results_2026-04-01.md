# test_ina_current — Results 2026-04-01

## Test Setup

| Parameter | Value |
|---|---|
| Current sensor | INA240A1 (×20 gain) |
| Shunt resistor | R100 (100 mΩ / 0.1 Ω) |
| Sensor output formula | V_out = I_load × 0.1 Ω × 20 = 2 V/A |
| Max measurable current | ~1.55 A (at 3.1 V ADC full scale) |
| ADC pin | GPIO13 (ADC1) |
| ADC config | 12-bit, 11 dB attenuation (0–3.1 V), 64× oversampling |
| Motor tested | JGB37-3650-2480, 169:1 gearbox |
| PWM frequency | 20 kHz |

> **Pin history:** Original test on XIAO ESP32-S3 used D3 (GPIO4). On the Waveshare ESP32-S3-Tiny, the INA240 output is on GPIO13.

## Wiring

High-side current sensing on the 24 V rail:

```
24V supply (+) ── INA240 IN+ ── shunt R100 ── INA240 IN- ── Motor Red (+24V)
24V supply (−) ── Motor Black ── GND (shared with ESP32)
INA240 VS  → 3.3 V
INA240 GND → GND
INA240 OUT → GPIO13
```

## Power-On Self-Calibration

The INA240 has a significant baseline offset (~720 mA equivalent) even at zero load. This is likely due to a combination of the INA240's input offset voltage, ADC nonlinearity at 11 dB attenuation, and the 3.3 V supply (unidirectional INA240 references output to GND, but any offset gets amplified by ×20).

The firmware performs automatic zero-calibration at boot:
1. Motor held at 0% PWM (must be stationary at power-on)
2. Takes 20 readings over 1 second (50 ms intervals, each 64× oversampled)
3. Averages to compute `currentOffsetMa`
4. Offset is subtracted from all subsequent readings

This makes the calibration invisible to the user and to Python scripts — all values reported via `READ`, `SWEEP`, `LOG`, and `STATUS` are already offset-corrected. The `[CAL]` debug messages at boot show the computed offset for diagnostic purposes.

This calibration approach compensates for temperature drift and component-to-component variation. It should be carried forward to all future firmware builds that use the INA240.

## Sweep Results (No Load, 169:1 Motor)

| PWM% | Motor RPM | Current (mA) |
|---|---|---|
| 0–35 | 0 | ~0 (dead band) |
| 40 | 680 | 13 |
| 50 | 2320 | 39 |
| 60 | 3740 | 60 |
| 70 | 5360 | 87 |
| 80 | 6880 | 125 |
| 90 | 8620 | 163 |
| 95 | 8560 | 162 |
| 100 | 8300 | 160 |

Peak no-load current draw: **~163 mA at 90% PWM**.

The current-vs-PWM curve is roughly linear in the active range (40–90%), consistent with the motor drawing more current as it spins faster (back-EMF and friction losses).

## Key Findings

1. **No-load current is very low.** Peak ~163 mA is well below any reasonable current budget. Under real mechanical load, current will be significantly higher — this can only be characterized once the end effector is assembled and driving actual fasteners.

2. **ADC noise is acceptable.** Readings fluctuate by ±5–10 mA, which is fine for current-limiting purposes. If needed, a simple moving-average filter can be added in firmware. For the intended use case (staying below a safety threshold with margin), this resolution is more than sufficient.

3. **Current tracks RPM closely.** Both curves plateau and slightly drop at 95–100% PWM, matching the motor saturation behavior seen in the feedback test. This confirms the INA240 is reading real motor current, not noise.

4. **Self-calibration works.** The ~720 mA raw offset is reliably zeroed at boot. Post-calibration readings at 0% PWM are within ±1 mA of zero.

## Integration Notes

For the final integrated firmware:
- Use the same power-on calibration routine (20 readings, 1 s)
- Current limiting strategy: set a threshold (e.g., 1000 mA) and reduce PWM when approaching it. Exact threshold depends on the robot's 24 V rail current budget and will be determined during loaded testing.
- The INA240A1 (×20 gain, 100 mΩ shunt) maxes out at ~1.55 A. If stall currents exceed this, consider the INA240A2 (×50) with a smaller shunt, or accept clipping at the safety limit.
