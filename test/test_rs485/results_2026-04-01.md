# test_rs485 — Results 2026-04-01

## Test Setup

| Parameter | Value |
|---|---|
| Transceiver | MAX3485 breakout module |
| Supply voltage | 3.3 V |
| TX pin | GPIO43 (board TX header) → module RXD (DI) |
| RX pin | GPIO44 (board RX header) → module TXD (RO) |
| DE/RE pin | GPIO18 → module DE + ~RE (tied) |
| Baud rate | 115200 |
| USB-RS485 adapter | Connected on COM7 |

> **Label confusion:** The MAX3485 module labels "TXD" and "RXD" from its own perspective. Module TXD (receiver output) connects to MCU RX (GPIO44). Module RXD (driver input) connects to MCU TX (GPIO43).

> **Pin history:** The original test on the XIAO ESP32-S3 used D6/D7/D0 (GPIO43/44/1). On the Waveshare ESP32-S3-Tiny, DE/RE moved to GPIO18; TX/RX remain on GPIO43/44.

## Tests Performed

### Test 1 — ESP32 → PC (transmit)
`SEND` command on USB-C serial → message received on USB-RS485 adapter. **PASS**

### Test 2 — PC → ESP32 (receive)
Message sent from USB-RS485 adapter → `[RX]` printed on USB-C serial. **PASS**

### Test 3 — Echo mode
Echo enabled → messages from RS-485 adapter are echoed back to adapter. `[RX]` and `[ECHO TX]` confirmed on USB-C serial. **PASS**

## Round-Trip Latency

Measured with `rs485_roundtrip.py` (50 trials per payload size).

### Initial test (50 ms inter-character timeout)

Firmware used a 50 ms timeout to detect end-of-message. This added a constant ~50 ms overhead to every response.

| Size (bytes) | Avg RTT (ms) | Min | Max |
|---|---|---|---|
| 1 | 51.5 | 50.8 | 52.1 |
| 64 | 63.6 | 63.1 | 64.4 |
| 256 | 96.7 | 95.9 | 97.4 |

Slope: 0.177 ms/byte — matches theoretical 2 × 0.0868 ms/byte perfectly. The variable component is pure wire time, with ~50 ms fixed overhead from the timeout.

### After switching to line-terminator parsing

Firmware changed to process messages on `\n`/`\r` instead of the 50 ms timeout. The fixed overhead dropped to ~1.5 ms.

| Size (bytes) | Avg RTT (ms) | Min | Max |
|---|---|---|---|
| 1 | 2.1 | 1.4 | 2.6 |
| 8 | 3.2 | 2.8 | 4.1 |
| 16 | 5.1 | 4.3 | 5.7 |
| 32 | 8.2 | 7.5 | 8.8 |
| 64 | 13.7 | 13.1 | 14.3 |
| 100 | 21.3 | 20.8 | 22.1 |
| 256 | 46.8 | 46.2 | 47.4 |

Fixed overhead: ~1.5 ms (ESP32 processing + DE/RE switching + USB driver latency).
Slope: 0.175 ms/byte — essentially the theoretical wire time for round-trip at 115200 baud.

## Key Findings

1. **RS-485 communication is fully functional.** Both directions work reliably with 0% error rate across 550 total trials.
2. **Latency is near-optimal.** After switching to line-terminator parsing, RTT is wire time + ~1.5 ms overhead.
3. **For a typical 16-byte command**, round-trip is ~5 ms — easily supports 100+ Hz control loops.
4. **Line-terminator framing is essential.** The 50 ms inter-character timeout approach added unacceptable latency. All future firmware should use `\n`-terminated message parsing.
5. **Zero errors** across all payload sizes (1–256 bytes) at 115200 baud.
